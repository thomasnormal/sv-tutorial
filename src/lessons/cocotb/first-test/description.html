cocotb (Coroutine-based Co-simulation Testbench) lets you write testbenches in Python rather than SystemVerilog. The simulator runs your DUT exactly as before; Python drives signals and checks outputs through an async coroutine.

A test is any async function decorated with @cocotb.test(). The dut argument gives you handles to every port — assign .value to drive, read .value to observe. await Timer(2, units="ns") suspends the coroutine until combinational logic settles — the Python equivalent of #2; in SystemVerilog.

The testbench in test_adder.py checks four input combinations. Open adder.sv and add the missing assign to make all checks pass.
