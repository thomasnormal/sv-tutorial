Every RTL engineer develops an ability to "see" hardware while reading code. Two constructs account for nearly all logic in a real design:

// Generic forms:
always_comb <out> = <sel> ? <a> : <b>;        // → multiplexer
always_ff @(posedge clk) <reg> <= <in>;       // → D flip-flop

Chain them and you get a pipeline stage — the fundamental unit of every datapath.

The gate-level view

Here is what mux_reg synthesises to. Every line of RTL maps to exactly one primitive:

always_ff → D flip-flop

Each variable assigned inside always_ff becomes a physical D flip-flop. The non-blocking assignment <= is the key: it means "latch this value at the clock edge." The triangle symbol on the DFF in the schematic marks the clock input.

always_comb → multiplexer

Each conditional assignment in always_comb becomes a multiplexer. The synthesis tool sees that d has two possible values and builds a 2:1 MUX. There is no storage — the output changes immediately when any input changes.

The logic [W-1:0] d wire between them is not a register — it is just a named connection. Only variables assigned in always_ff become flip-flops.
