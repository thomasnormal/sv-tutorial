The most common synthesis surprise for new RTL engineers is the inferred latch. Write an if or case in always_comb without covering every possible input value, and the synthesiser adds a latch to hold the output when the uncovered case occurs:

always_comb begin
  case (sel)
    2'b00: y = 4'b0001;
    2'b01: y = 4'b0010;
    2'b10: y = 4'b0100;
    // 2'b11 missing → y must HOLD its value → synthesiser adds a latch
  endcase
end

Latch vs flip-flop

Two fixes

1. Cover all cases. Add the missing branch or a default assignment. The synthesiser sees that y always gets a value — no need to hold anything — so no latch is generated:

case (sel)
  2'b00: y = 4'b0001;
  ...
  2'b11: y = 4'b1000;   // or: default: y = '0;
endcase

2. Add unique. The unique case keyword tells the synthesiser that exactly one branch will match — it can build a parallel MUX tree instead of a priority chain, which is smaller and faster:

unique case (sel)   // ← one keyword, better hardware
  ...
endcase

In simulation, unique case also enables a runtime check: if two branches ever match simultaneously, or none matches, the simulator reports an error. This catches bugs the plain case would silently ignore.
