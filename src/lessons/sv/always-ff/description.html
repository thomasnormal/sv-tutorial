always_ff models edge-triggered sequential logic. The flip-flop array inside an SRAM is the memory — every bit stored in the chip lives in a flip-flop that updates on each rising clock edge.

The sensitivity list @(posedge clk) means the block re-evaluates only when the clock transitions from 0 → 1 — the rising edge. Between edges, every register holds its value unchanged:

Every port must declare a direction: input (driven from outside this module), output (driven from inside), or inout (bidirectional, rare). The keyword after the direction — logic — is the signal type. Open sram_core.sv to see each port annotated with its purpose.

Use non-blocking assignment (<=) inside always_ff. All right-hand sides are evaluated first, then all assignments happen simultaneously — this prevents races and models real flip-flop behaviour.

Open sram_core.sv and fill in the always_ff body with two statements:

  • When we is high, write wdata into mem[addr]
  • Always register the read: capture mem[addr] into rdata

The read is registered: drive addr on cycle N and rdata reflects that address on cycle N+1. This is the standard synchronous-read SRAM model.

tb.sv writes three values to addresses 2, 7, and 0, then reads them back one cycle later. Each read prints PASS or FAIL — run it before solving to see all three fail, then again after to confirm they all pass. Open the Waves tab to see clk, we, addr, wdata, and rdata over time.
