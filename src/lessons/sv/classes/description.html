A class bundles data and methods into a single unit:

class <name>;
  <field declarations>
  function new(<args>);  // constructor
    <initialization>
  endfunction
  function string convert2string(); ... endfunction
endclass

Classes are the foundation of UVM — every sequence item, driver, and monitor extends uvm_object or uvm_component, which are themselves classes.

Handle semantics

A class variable is a handle, not a value. Assigning one handle to another makes both variables point to the same object:

mem_item t1 = new(1, 4'd5, 8'hA5);
mem_item t2 = t1;       // t2 and t1 share the object
t2.wdata = 8'hFF;
$display(t1.wdata);     // prints FF — same object!

To get an independent copy, call new(...) again:

t2 = new(0, 4'd3, 8'h00);  // t2 now points to a different object
// t1.wdata is still 8'hFF

The this keyword

Inside a constructor, argument names often shadow field names. Use this.field to refer to the field unambiguously:

function new(bit we, logic [3:0] addr, ...);
  this.we   = we;    // 'this.we' is the field; 'we' is the argument
  this.addr = addr;
endfunction
