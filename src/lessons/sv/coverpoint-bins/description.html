For the SRAM we care about exercising both halves of the address space — not just "did any address get hit." Named bins let you group values into meaningful categories:

coverpoint addr {
  bins lo_half = {[0:7]};    // group: lower 8 addresses
  bins hi_half = {[8:15]};   // group: upper 8 addresses
}

Now coverage is measured per group, not per raw value. You can also exclude values you never intend to exercise with ignore_bins:

ignore_bins reserved = {14, 15};  // these addresses are off-limits

Open cov_bins.sv. The cp_we coverpoint already has explicit bins. Add explicit bins to cp_addr:

  • A bin for the lower half of the address space (0–7)
  • A bin for the upper half (8–15), but exclude addresses 14 and 15 as reserved using ignore_bins

There is also illegal_bins — like ignore_bins it is excluded from coverage, but it additionally causes a simulation error if the value ever occurs. Use it for encodings that should be impossible, not just uninteresting.
