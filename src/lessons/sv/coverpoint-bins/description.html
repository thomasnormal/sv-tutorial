Coverpoints can be partitioned into named bins so you can track meaningful categories instead of raw values.

You can also exclude uninteresting values with ignore_bins:

coverpoint burst {
  bins low = {0,1};
  bins high = {2};
  ignore_bins idle = {3};
}

Open cov_bins.sv and add the two bins and one ignore_bins entry.

Use bins to encode intent. A reviewer should be able to infer the verification goal from bin names alone.
