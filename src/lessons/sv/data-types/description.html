The SRAM you are building has two very different kinds of code: the RTL that will be synthesized into silicon, and the testbench that exercises it in simulation. SystemVerilog gives each a different family of types. Using the wrong one either hides bugs or causes unexpected behavior.

4-state vs 2-state

RTL signals use logic — a 4-state type. The extra two states are:

  • X — unknown: the signal is uninitialised, or two drivers are fighting. X propagates through logic gates, making the bug impossible to miss in the waveform or log.
  • Z — high-impedance: no driver is connected. Mainly used on tri-state buses and chip I/O pads.

Testbench code uses 2-state types. They only hold 0 and 1, initialise to 0 automatically, and support signed arithmetic:

Type Width Signed? Typical use
bit1unsignedsingle-bit flags
int32signedloop counters, error counts
byte8signedsmall signed values
longint64signedtimestamps, cycle counts

Using bit [31:0] instead of int for a counter seems harmless — until you set it to -1 and count < 0 silently returns false because bit is unsigned. Using bit [7:0] instead of logic [7:0] for an RTL port means X silently becomes 0, hiding uninitialised-signal bugs.

Packed vs unpacked arrays

The bracket placement determines what you get:

logic [7:0] x      // packed vector: x[0] is 1 bit (bit-select)
logic [7:0] x [4]  // unpacked array: x[0] is 8 bits (element-select)

Packed brackets go before the name — they form one contiguous bit-vector. Unpacked brackets go after the name — each element is independent storage. The SRAM memory array logic [WIDTH-1:0] mem [0:DEPTH-1] is exactly this: addr selects an element; [WIDTH-1:0] is each element's packed width.

Exercise

Open data_types.sv. Three declarations use the wrong type — each causes one test to fail. Fix them:

  • count is bit [31:0] — unsigned, cannot hold -100. Change to int.
  • data is bit [7:0] — 2-state, silently drops X to 0. Change to logic [7:0].
  • mem is logic [7:0] mem — a scalar; mem[0] is just bit 0. Add [4] after mem to make it an unpacked array of four bytes.

Run before and after — three FAIL lines should become three OK lines, then PASS.
