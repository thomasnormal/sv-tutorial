fork...join spawns parallel threads inside a procedural block. The three variants control when the parent thread continues:

fork
  <thread 1>;
  <thread 2>;
  <thread 3>;
join          // parent resumes when ALL threads finish
join_any      // parent resumes when the FIRST thread finishes
join_none     // parent resumes immediately; threads run in background

Each entry in the fork block is an independent thread. Tasks marked automatic get their own local stack — without automatic, all calls share the same variables.

Choosing the variant

Variant Parent resumes when… Typical use
joinall threads doneDrive multiple signals simultaneously
join_anyfirst thread doneTimeout: race stimulus vs. watchdog
join_noneimmediatelyBackground monitors, clock generators

  1. Add a fork...join_any block with the same three delay_print calls. After it, print "first done at <time> ns". Expected: continues 10 ns after the fork.
  2. Add a fork...join_none block that launches one 20 ns delay_print. Immediately after, print "parent continues at <time> ns" — this should print before the background thread fires.
