Static arrays have a fixed size set at compile time. Testbenches need structures that grow at runtime — enter dynamic arrays and queues.

Dynamic arrays

type name[];       // declared, but empty (null handle)
name = new[n];     // allocate n elements — all zeroed
name.size()        // returns current length
name[i]            // index access, same as static array

Queues

type name[$];          // unbounded queue, initially empty
name.push_back(v);     // append v to the tail
name.pop_front();      // remove and return the head element
name.size()            // number of elements currently in the queue

Queues are the standard data structure for scoreboards and FIFOs in UVM environments — the monitor pushes transactions onto the queue; the scoreboard pops and checks them.

Open addr_buf.sv and complete four tasks:

  1. Declare the dynamic array at module scope: logic [3:0] addrs[]
  2. Inside initial, allocate 8 elements: addrs = new[8]
  3. Push the first four elements of addrs into q using push_back.
  4. Pop all elements from q using pop_front inside a while loop, printing each value.

The test passes when q.size() == 0 after all pops.
