Sometimes an in-progress property should be terminated early — either because an abort condition renders the check irrelevant, or because its occurrence should itself be flagged as a failure. SVA provides two operators for this:

  • reject_on(cond) prop — if cond becomes true while prop is being evaluated, fail immediately.
  • accept_on(cond) prop — if cond becomes true, pass immediately (abort without failure).

Both have synchronous variants — sync_reject_on and sync_accept_on — that only sample cond at the property's clock edge instead of asynchronously.

Example: a burst transfer should normally complete with ack within 4 cycles, but if an error fires mid-burst the property should immediately fail:

start |=> reject_on(err) (##[1:4] ack)

Open abort_check.sv and complete the property body using reject_on.

reject_on(cond) seq is equivalent to (!cond) throughout seq — both fail when cond goes true during the sequence window. reject_on is more expressive because it also applies to composite properties, not just sequences.
