A checker is an encapsulated assertion block that can be instantiated like a module. The key benefit is reuse: write the property once, instantiate it for every signal pair in the design.

checker handshake_check(input logic clk, req, ack);
  property p;
    @(posedge clk) req |=> ##[1:3] ack;
  endproperty
  req_ack_a: assert property (p);
endchecker

module top(input logic clk, req1, ack1, req2, ack2);
  // Same check applied to two independent channels
  handshake_check c1(clk, req1, ack1);
  handshake_check c2(clk, req2, ack2);
endmodule

Unlike a module, a checker is automatically excluded from synthesis tools — no risk of accidentally instantiating it in RTL. It can also accept sequence, property, and event typed formal arguments, making it a fully parameterisable assertion bundle — the SVA equivalent of generic verification IP.

The WASM tool does not yet lower the checker/endchecker keywords. This exercise uses module instead — the properties and assertions are identical; only the encapsulation keyword differs.
