In the previous chapter you ran concurrent assertions driven by a testbench — the simulator only checked the paths you drove. A model checker goes further: it exhaustively explores every possible input sequence up to a bounded depth, finding counterexamples without any testbench at all. The lessons from here on use CIRCT's bounded model checker (BMC).

Before concurrent assertions, there are immediate assertions — non-temporal checks that live inside a procedural block. They fire the instant the simulator reaches them, like a runtime sanity check:

assert (expression)
  pass_action;
else
  fail_action;

If the expression is 0, x, or z, the assertion fails and the else branch runs.

Open fifo_checker.sv and add two assertions inside the always @(posedge clk) block — one that prevents writing to a full FIFO, and one that prevents reading from an empty FIFO. Express each condition as a Boolean that must always be true.

Use $error in the else clause for an automatic timestamp and file/line location in the log.
