SystemVerilog's logic type has four states: 0, 1, X (unknown), and Z (high-impedance). The X state propagates through logic: any gate with an unknown input produces an unknown output. Uninitialized SRAM memory is a classic source — reading a location before it has been written returns X, which then flows into the rest of the design.

$isunknown(expr)

$isunknown(expr) returns 1 if any bit of expr is X or Z. It is simulation-only — formal tools use two-state logic where X and Z do not exist. Use it to catch:

  • Uninitialized registers (start of simulation before reset completes)
  • Floating bus wires (undriven output ports)
  • SRAM reads before any write has occurred

Negate it to express a positive assertion — the signal must always be known:

!$isunknown(signal)

Wrap it in a concurrent property with disable iff (!rst_n) to suppress the check during reset, when everything is legitimately undriven:

property p_no_x_signal;
  @(posedge clk) disable iff (!rst_n)
    !$isunknown(signal);
endproperty
assert property (p_no_x_signal);

Exercise

Open xcheck.sv. The module monitors three SRAM signals — we, addr, and rdata — and must assert that none of them ever carries X or Z after reset. Fill in the three property bodies:

  • TODO 1: we must never be X or Z
  • TODO 2: addr must never contain X or Z
  • TODO 3: rdata must never contain X or Z

The testbench deliberately drives each signal to 'x in turn. Run the simulation to see the assertion fire at each injection point.

$isunknown is simulation-only. Formal tools work in two-state (0/1) logic and do not model X or Z — do not use it inside a BMC or LEC design.
