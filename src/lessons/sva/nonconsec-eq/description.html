The non-consecutive equal repetition operator [=m] requires a boolean to be true exactly m times — with the same flexibility as [->m] regarding gaps, but with one key difference at the end:

  • sig[->m] — sequence ends at the m-th true cycle
  • sig[=m] — sequence ends at any later cycle after the m-th true cycle

This matters when the signal that follows (done, ready, …) can arrive on the same cycle as the last match or several cycles after it:

// [->3]: done must be the cycle immediately after the 3rd ack
req |=> ack[->3] ##1 done

// [=3]: done may arrive at any point on or after the 3rd ack
req |=> ack[=3] ##[0:$] done

Open multi_ack2.sv and complete the property body using ack[=3] ##[0:$] done.

Use [=m] when the protocol says "exactly m pulses must occur before completion" but the completion signal has variable latency. Use [->m] when completion is defined as arriving one cycle after the final pulse.
