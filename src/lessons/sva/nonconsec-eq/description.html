The non-consecutive equal repetition operator [=m] looks like [->m] but has a different rule about when the sequence ends:

Open multi_ack2.sv. The spec is: after start, ack must pulse exactly 3 times, and then done must go high at some point afterwards — but not necessarily on the very next cycle.

Complete the property using [=3] and an unbounded delay to done.

Use [=m] when the protocol says "m pulses must occur before completion" but the completion signal has variable latency. Use [->m] when completion is defined as arriving one cycle after the final pulse.
