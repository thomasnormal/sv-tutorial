A recursive property is a property that refers to itself in its own body, creating a self-referential temporal statement. They must use the non-overlapping implication |=> (never |->) to avoid zero-time loops:

property p_hold(logic sig, logic release_cond);
  @(posedge clk) sig && !release_cond |=> p_hold(sig, release_cond);
endproperty

This reads: "if sig is high and release_cond is not, then one cycle later the same condition must hold again." The recursion unrolls until release_cond becomes true.
