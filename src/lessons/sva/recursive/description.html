Some guarantees must hold for all time — not just for a fixed number of cycles. A recursive property expresses this by referring to itself in its consequent:

property p_lock_hold;
  @(posedge clk) lock && !unlock |=> p_lock_hold;
endproperty

This reads: "if lock is high and unlock has not fired, one cycle later the same condition must hold again — repeating indefinitely until unlock fires." The recursive call must appear in the consequent (never the antecedent), and must use |=> to avoid a zero-time loop.

Full formal tools prove recursive properties by induction: prove (a) the property holds at step 0, and (b) if it holds at step k, it holds at step k+1. Together, these give an unbounded proof.

The WASM tool does not yet support recursive property calls. This exercise uses the equivalent one-step inductive form: lock && !unlock |=> lock. BMC checks this for every depth up to its bound — which is equivalent to the recursive form within any finite trace.
