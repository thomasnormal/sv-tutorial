A named sequence can declare formal arguments — typed placeholders that are substituted with actual values at every call site:

sequence <name>(<type> <arg1>, <type> <arg2>);
  <body using arg1 and arg2>
endsequence

Without formals, checking two addresses means duplicating the body:

sequence write_at_0;  we && addr == 4'd0 && wdata == 8'hA5 endsequence  // ↑ same structure
sequence write_at_15; we && addr == 4'hF  && wdata == 8'hFF endsequence  // ↑ copy-pasted

One template serves both. sram_check.sv defines two complementary sequences with typed formals (logic [3:0] a, logic [7:0] d):

  • write_txn(a, d) — fires when a write of d to address a occurs
  • read_txn(a, d) — fires when addr == a and rdata == d are simultaneously true

Both are wired together in wr_rd_p. The timing:

Open sram_check.sv and complete four tasks:

  1. Fill in the write_txn body using formals a and d.
  2. Fill in the read_txn body: addr equals a and rdata equals d.
  3. Fill in the wr_rd_p property body using write_txn, |=>, and read_txn.
  4. Write two labelled assert property statements for (4'd0, 8'hA5) and (4'hF, 8'hFF).

Formals are inlined at elaboration time — the model checker sees two fully expanded, independent properties. To check all address/data pairs in one pass, you need a local variable to capture the write's values and carry them into the consequent (covered in Local Variables in Sequences).
