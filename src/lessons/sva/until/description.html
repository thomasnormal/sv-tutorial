The until family of operators expresses "hold this condition while waiting for that one":

  • p until q — weak: p holds at every tick until q becomes true. If q never occurs the property still holds (vacuously).
  • p s_until q — strong: same as above but q must eventually occur.
  • p until_with q — inclusive: p is also required to hold on the same cycle that q first becomes true.

A common use: a valid signal must stay high until a done pulse arrives:

property p_valid_holds;
  @(posedge clk) valid |-> valid s_until done;
endproperty

Open hold_check.sv and complete both properties: one using s_until and one using until_with.

The nexttime operator is the single-step companion: nexttime p requires p to hold at the very next clock tick. s_nexttime p is the strong variant (fails if there is no next tick). These combine naturally with until to build recursive temporal patterns.
