An implication property antecedent |-> consequent passes vacuously when the antecedent never evaluates to true. The assertion fires zero times and reports no failures — which can be falsely reassuring.

The testbench below never drives req high, so $rose(req) never fires. The assert reports a clean pass, but nothing was actually checked.

The fix is to always pair an assert with a companion cover property on the same property. cover does not pass vacuously: its pass action only fires when the antecedent actually matched and the full sequence succeeded:

rg_assert: assert property (req_gnt)
  else $display("FAIL at t=%0t", $time);
rg_cover:  cover  property (req_gnt)
           $display("antecedent fired at t=%0t", $time);

If the simulation ends with zero cover hits and zero assert failures, the property has told you nothing.

A common practice: suppress the assert pass action entirely (leave it empty) and rely on the companion cover to confirm the property was exercised. The assert only needs to fire when something goes wrong.
