Random stimulus without a stopping condition is unpredictable — you may run far more cycles than needed, or stop before all bins are hit. Coverage-driven verification closes this loop: the test keeps generating stimulus until coverage reaches its target, then stops.

The pattern uses get_coverage() on the covergroup to query current progress:

task run_phase(uvm_phase phase);
  phase.raise_objection(this);
  while (env.cov.adder_cg.get_coverage() < 100.0) begin
    adder_seq seq = adder_seq::type_id::create("seq");
    seq.start(env.agent.seqr);
    `uvm_info("TEST", $sformatf("coverage = %.1f%%",
      env.cov.adder_cg.get_coverage()), UVM_LOW)
  end
  phase.drop_objection(this);
endtask

Open adder_test.sv and replace the single seq.start() call with the while loop above.

Always add a maximum-iteration guard to prevent infinite loops when a coverage bin is unreachable. A good practice: if coverage has not improved after N batches, print a warning and break — then investigate which bin is stuck and why.
