Individual coverpoints track each signal independently. Cross coverage tracks combinations — it creates a bin for every pair of values from two coverpoints and checks that each combination was seen:

cp_a: coverpoint item.a { bins lo = {[0:85]}; bins mid = {[86:170]}; bins hi = {[171:255]}; }
cp_b: coverpoint item.b { bins lo = {[0:85]}; bins mid = {[86:170]}; bins hi = {[171:255]}; }

ax: cross cp_a, cp_b;

The cross automatically generates 3×3 = 9 bins: <lo,lo>, <lo,mid>, …, <hi,hi>. All nine must be hit to reach 100%.

Open adder_coverage.sv. The two coverpoints are already written. Add the ax: cross cp_a, cp_b; line inside the covergroup.

Cross coverage can grow explosively with more coverpoints or wider bins. It is common to exclude impossible or uninteresting combinations with ignore_bins or illegal_bins to focus coverage effort where it matters.
