Writing verification code that pokes raw hex addresses is fragile — change the register map and everything breaks. The Register Abstraction Layer (RAL) replaces raw addresses with named, typed register objects:

// Before: error-prone
axi.write(32'h4000_0010, 8'h05);   // Which register? Which field?

// After: self-documenting
ral.ctrl.enable.set(1);
ral.ctrl.mode.set(2'b10);

The UVM RAL skeleton

Every register file follows the same two-class pattern:

class <RegName> extends uvm_reg;
  uvm_reg_field <field>;
  function new(string name="<RegName>"); super.new(name, <width>, UVM_NO_COVERAGE); endfunction
  virtual function void build();
    <field> = uvm_reg_field::type_id::create("<field>");
    <field>.configure(this, <size>, <lsb_pos>, "RW", 0, 0, 1, 0, 1);
  endfunction
  `uvm_object_utils(<RegName>)
endclass

class <BlockName> extends uvm_reg_block;
  <RegName> <reg>;
  function new(string name="<BlockName>"); super.new(name, UVM_NO_COVERAGE); endfunction
  virtual function void build();
    <reg> = <RegName>::type_id::create("<reg>");
    <reg>.build();
    <reg>.configure(this);
    default_map = create_map("map", 'h0, 1, UVM_LITTLE_ENDIAN);
    default_map.add_reg(<reg>, '<offset>, "RW");
    lock_model();
  endfunction
  `uvm_object_utils(<BlockName>)
endclass

The three map calls at the end of build():

  • create_map() — creates a named address map; the 3rd argument is bus width in bytes, the 4th is endianness
  • add_reg() — places a register at a byte offset within the map
  • lock_model() — freezes the model structure; must be called before any register access

UVM RAL hierarchy

The starter model

This lesson ships a hand-rolled register model that already compiles and simulates in MOX. Study how it maps to the skeleton above before converting it:

  • Fields as class members — logic enable and logic [1:0] mode are plain variables; uvm_reg_field replaces them
  • to_data() — packs fields by hand: {5'b0, mode, enable}; uvm_reg::get() does this automatically using the lsb_pos from configure()
  • ADDR localparam — the address is baked in; uvm_reg_block owns the map instead

The configure() call explained

uvm_reg_field::configure() takes 9 parameters. The most important:

ParameterValue in this lessonMeaning
parentthisThe owning uvm_reg
size1 / 2Width in bits
lsb_pos0 / 1Position of the LSB in the register — replaces the bit concatenation in to_data()
access"RW"Access policy (RW, RO, WO, …)
reset0Reset value
